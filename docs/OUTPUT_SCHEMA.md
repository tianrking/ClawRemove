# Machine-Consumable JSON Output Contract

ClawRemove produces stable JSON output natively when the `--json` flag is supplied. This output schema is designed to be easily parsed by upper-computer systems, local orchestration agents, and future GUI wrappers.

The JSON schema strictly mirrors the internal `model.Report` struct.

## Root Schema Properties

When running commands like `audit`, `plan`, `apply`, `verify`, or `explain`, the root JSON object will always contain the following keys:

```json
{
  "ok": true,
  "product": "openclaw",
  "command": "verify",
  "dryRun": true,
  "auditOnly": true,
  "host": {
    "os": "darwin",
    "arch": "arm64"
  },
  "capabilities": {
    "skills": [],
    "tools": []
  },
  "discovery": { ... },
  "evidence": { ... },
  "verify": { ... },
  "plan": { ... },
  "results": [],
  "advice": { ... }
}
```

- **`ok`**: Boolean indicating overall success. If false, an execution or engine fault occurred.
- **`command`**: The CLI action currently being operated (e.g., `apply`).
- **`dryRun` / `auditOnly`**: Booleans indicating if destructive changes were bypassed.

## `discovery`

The `discovery` block contains raw, deterministic facts collected from the file system, process tree, and service managers *before* confidence rules are applied.

```json
  "discovery": {
    "platform": "darwin",
    "homeDir": "/Users/someone",
    "stateDirs": [
      "/Users/someone/.openclaw"
    ],
    "services": [
      {
        "platform": "darwin",
        "scope": "user",
        "name": "ai.openclaw.gateway"
      }
    ],
    "processes": [],
    "packages": []
...
  }
```

## `verify`

The `verify` block categorizes evidence against actual host paths and groups them into specific buckets for review.

```json
  "verify": {
    "verified": true,
    "summary": {
      "exact": 1,
      "strong": 0,
      "heuristic": 0
    },
    "residuals": [],
    "confirmed": [
      {
        "kind": "stateDir",
        "target": "/Users/someone/.openclaw",
        "evidence": "exact",
        "reason": "Exact path match in facts",
        "risk": "medium",
        "rule": "exact-state-dir",
        "source": "filesystem",
        "confidence": 1.0
      }
    ],
    "investigate": []
  }
```

- **`confirmed`**: Items that have `confidence >= 0.8` (e.g., exact or strong matches). These are safely removable.
- **`investigate`**: Items with `confidence < 0.8` (e.g., fuzzy name matches, vague shell hooks).

## `plan`

The `plan` block contains the concrete actions generated by the engine. 

```json
  "plan": {
    "actions": [
      {
        "kind": "remove_path",
        "target": "/Users/someone/.openclaw",
        "reason": "Exact path match in facts",
        "evidence": "exact",
        "rule": "exact-state-dir",
        "source": "filesystem",
        "confidence": 1.0,
        "risk": "medium"
      }
    ]
  }
```

- **`kind`**: Enumerated strings such as `remove_path`, `run_command`, `edit_file`, or `report_only`.
- **`report_only`**: Destructive execution was blocked because of the `--dry-run` flag or low confidence. Action should not be mechanically forced unless visually overridden.

## `results`

If the `command` is `apply`, the `results` array will map one-to-one to the actions executed, showing the final status.

```json
  "results": [
    {
      "ok": true,
      "action": "remove_path",
      "target": "/Users/someone/.openclaw",
      "reason": "Deleted successfully",
      "dryRun": false
    }
  ]
```

## `advice` (Optional)

If `--ai` or `explain` was invoked, the `advice` block will be populated. It contains human-friendly orchestrator recommendations.

```json
  "advice": {
    "mode": "controlled",
    "authority": "advisory-only",
    "thoughtSummary": "Review services before path cleanup.",
    "recommendations": [
      {
        "kind": "review_services",
        "target": "1 matching service",
        "reason": "Persistent service found.",
        "risk": "medium",
        "optIn": false,
        "evidence": "strong"
      }
    ],
    "riskNotes": [],
    "userMessage": "Unload persistent services first."
  }
```
